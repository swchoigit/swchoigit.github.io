## 클래스
* 객체를 생성하기 위한 '설계도' 또는 '틀'
* 객체의 속성(변수)과 행위(메서드)를 정의해 놓은 추상적인 개념
* 그 자체로는 메모리를 차지하거나 직접 일을 수행하지 않음

### 클래스 변수
* `static` 키워드가 붙어 선언 (예외: Python에서는 키워드 없이 클래스 블록 바로 아래 선언)
* 해당 클래스로 생성된 모든 인스턴스가 하나의 메모리 공간을 공유
* 인스턴스를 생성하지 않고도 클래스 이름을 통해 직접 접근 가능
* 프로그램이 시작되어 클래스가 메모리에 로드될 때 생성되고 프로그램 종료 시까지 유지

## 인스턴스
* 클래스(설계도)를 바탕으로 메모리에 실제 구현된 구체적인 실체(객체)
* 프로그램이 실행되는 동안 고유한 상태를 유지하며 실질적인 작업을 수행
* 하나의 클래스로부터 여러 개의 독립적인 인스턴스를 생성 가능

### 인스턴스 변수
* 클래스 영역에 선언되며, 객체(인스턴스)가 생성될 때마다 메모리에 개별적으로 생성
* 각 인스턴스마다 독립적인 저장 공간을 가지므로 서로 다른 값을 유지 가능
* 객체가 메모리에서 소멸할 때 함께 소멸
